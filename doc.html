Editor Python - Documentação Técnica

O editor opera em três modos distintos, determinados pela presença de imports específicos no código. A detecção ocorre em main.js na seguinte ordem: primeiro verifica import de sql, depois tkinter. Caso nenhum seja encontrado, executa no interpretador Skulpt.

Modo Critério de ativação Arquivo responsável
Python (Skulpt) Ausência de imports sql/tkinter main.js
Tkinter import tkinter ou from tkinter import tk.js
SQL import sql ou from sql import sql.js

1. Modo Python (Skulpt)

Executa código Python via Skulpt, interpretador implementado em JavaScript. Suporta sintaxe Python 3 com as limitações inerentes ao Skulpt.

Funcionalidades

  • print() - saída direcionada ao console da aplicação
  • input() - abre campo de entrada no console, resolução via Promise
  • Indentação com Tab (insere 4 espaços)
  • Atalho Ctrl+Enter para execução
  • Bibliotecas padrão disponíveis no Skulpt (math, random, etc.)

Exemplo

# Cálculo de média
notas = []
for i in range(3):
    nota = float(input(f"Nota {i+1}: "))
    notas.append(nota)

media = sum(notas) / len(notas)
print(f"Média: {media:.2f}")

if media >= 7:
    print("Aprovado")
else:
    print("Reprovado")

2. Modo Tkinter

Não executa Tkinter de fato. O código é analisado linha a linha por tk.js e elementos da interface são gerados como HTML dentro de um iframe.

Componentes suportados

  • Tk() + .title() - gera container com barra de título
  • Label() - div com texto
  • Entry() - input type="text"
  • Button() - botão HTML com evento de clique
  • .pack() - layout vertical (side não implementado)
  • .get() - recupera valor de Entry
  • .config(text=...) - atualiza texto de Label

Funções e escopo

Funções definidas com def são armazenadas e podem ser executadas via command de botões. O parser possui um interpretador simplificado para blocos de função com suporte a:

  • Atribuição de variáveis (n1 = float(v1.get()))
  • Chamada entre funções
  • print() - saída no mini-console integrado
  • try/except - tratamento básico de erros
  • limpar() - função customizada que limpa labels e entries
  • f-strings com expressões (f"{n1} + {n2} = {n1 + n2}")

Exemplo

from tkinter import *

root = Tk()
root.title("Calculadora")

v1 = Entry()
v1.pack()
v2 = Entry()
v2.pack()
resultado = Label(text="Resultado: ")
resultado.pack()

def somar():
    try:
        n1 = float(v1.get())
        n2 = float(v2.get())
        resultado.config(text=f"Soma: {n1 + n2}")
    except:
        resultado.config(text="Erro")

Button(text="Somar", command=somar).pack()
root.mainloop()
Limitação: O parser não implementa orientação a objetos, herança ou métodos complexos. É uma tradução superficial baseada em padrões de código.

3. Modo SQL

Implementa um banco de dados em memória via JavaScript. Comandos SQL são parseados e executados contra estruturas de dados internas.

Comandos suportados

  • CREATE TABLE nome (coluna1, coluna2, ...) - define colunas (tipos ignorados)
  • INSERT INTO nome VALUES (val1, val2, ...) - valores podem ter ou não aspas
  • SELECT colunas FROM nome [WHERE coluna = valor] - where apenas igualdade
  • UPDATE nome SET coluna = valor WHERE condição
  • DELETE FROM nome WHERE condição
  • DROP TABLE nome

Tratamento de erros

O parser inclui validação específica para vírgulas ausentes em CREATE TABLE, gerando mensagem com ponteiro indicando a posição aproximada do erro.

Exemplo

import sql

CREATE TABLE alunos (id, nome, nota)
INSERT INTO alunos VALUES (1, "João", 8.5)
INSERT INTO alunos VALUES (2, "Maria", 9.0)

SELECT * FROM alunos
SELECT nome FROM alunos WHERE nota = 9.0

Interface do Editor

Componentes

Elemento Função
Editor Área contenteditable com suporte a tab (4 espaços)
Executar Executa o código (atalho Ctrl+Enter / Cmd+Enter)
Docs Abre esta documentação
Limpar Console Limpa a área de saída
Exemplos Abre painel com exemplos pré-definidos (SQL e Tkinter)
Status Indica estado atual: "carregando", "executando", "pronto"
Console Área de saída. No modo Tkinter, substituído por iframe com a GUI

Painel de Exemplos

O botão "Exemplos" exibe um painel flutuante com duas opções:

  • SQL Example - carrega script de criação e consulta de tabela
  • Tkinter Example - carrega exemplo da calculadora

O painel é posicionado dinamicamente abaixo do botão e fechado ao clicar fora.

Comportamento dos modos

Modo Saída Comportamento
Skulpt Console principal Execução real de Python
Tkinter Iframe + mini-console Tradução para elementos HTML
SQL Tabelas formatadas em texto Operações em banco fictício
Observação: Os modos são mutuamente exclusivos. A presença de import sql tem precedência sobre tkinter devido à ordem de verificação no código.
